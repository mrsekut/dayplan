# Dayplan リビルド: ヒアリングメモ

## 核心の目的

**タスク間の意思決定コストをゼロにして、今のタスクだけに集中できる状態を作る**

## 根本問題

スケジュールが静的で現実に追従できず、形骸化する。

```
計画 → 差し込み → リスケ面倒 → 乖離 → 無視 → 意思決定コスト削減が達成できない
```

## 課題一覧

| # | 課題 | 補足 |
|---|------|------|
| A | リスケの摩擦が大きい | 「15分延長」等のワンアクション操作で後続が自動調整されてほしい |
| B | タスク完了の明示的マークがない | 自動切替だと無視できる。完了→次タスク提示のループが必要 |
| C | 通知に強制力がない | PiPは見えるが行動を変えない。MeetingBar的な画面占有が欲しい |
| D | 細かいタスクの居場所がない | Beadsに統一し、グルーピングしてスケジュールにマッピングしたい |
| E | 振り返りが手動 | 自動化されれば嬉しいが優先度低 |

## 設計上の前提

- **ツール分担**: Beads(タスク作成) → Strand(AI自動実行) → Dayplan(グルーピング+スケジュール)
- **GCal**: MCP経由で読み取り。dayplanへの組み込み不要（AIが理解すれば十分）
- **CLI必須**: AI操作のため。人間向けUIは形式問わず（Web/TUI/デスクトップアプリ）
- **AI向けインターフェース**: 現dayplanの良い点。維持する

## 現dayplanの良い点

- AI向けのインターフェースになっている（CLIで操作、prime/onboardコマンド等）
- カレンダービューで時間感覚が掴める

## 粒度とグルーピング

- 5-10分の細かいタスクはBeadsに投げる
- それらをグルーピングして「30分: 雑務消化」のようなブロックにまとめたい
- 思考系タスクと作業系タスクを分けてマッピングしたい

## リスケの理想形

- 「15分延長」のようなワンアクション操作
- 延長すると後続タスクが自動で調整される
- 理想: 画面キャプチャで脱線検知→自動中断（将来的）

## タスク完了フロー

- 完了を明示的にマークする（自動切替だと無視してしまう）
- 未マークのまま時間オーバー → AIがリスケ or 通知で強制介入
- 完了マーク → 次タスクが提示される

## 通知の理想形

- MeetingBar的な画面占有型の通知（5分前など）
- PiPは見えるが行動を変えるほどの強制力がなかった
- タスク間にバッファがあった方がいいかもしれない
